# 聊天页面响应式设计说明

## 概述

聊天页面已实现完整的响应式设计，支持桌面端和移动端设备。

## 主要改进

### 1. 移动端侧边栏（Sheet 抽屉）

- **桌面端（≥768px）**：侧边栏固定显示，可调整宽度
- **移动端（<768px）**：侧边栏隐藏，通过汉堡菜单按钮打开抽屉式侧边栏
- 选择助手或会话后自动关闭侧边栏，提升用户体验

### 2. 移动端顶部导航栏

- 固定在页面顶部（`fixed top-0`）
- 包含：
  - 汉堡菜单按钮（打开侧边栏）
  - 当前页面标题
  - 快捷导航按钮（Dashboard、Chat）
- 高度：52px，主内容区域添加 `pt-[52px]` 避免被遮挡

### 3. 桌面端导航栏

- 垂直布局的左侧导航栏（ChatNavRail）
- 在移动端完全隐藏（`hidden md:block`）
- 保持原有的图标 + Tooltip 交互

### 4. 会话头部优化

- **模型选择器**：
  - 桌面端：220px 宽度
  - 移动端：140px 宽度
- **按钮尺寸**：
  - 桌面端：h-9 w-9
  - 移动端：h-8 w-8
- **Bridge 按钮**：移动端隐藏（`hidden md:flex`）
- **模型信息**：移动端隐藏副标题（`hidden md:block`）

### 5. 侧边面板（Eval Panel / Bridge Panel）

- **桌面端**：固定宽度 384px（w-96）
- **移动端**：全屏宽度（w-full）
- 保持从右侧滑入的动画效果

## 技术实现

### 使用的 Tailwind 响应式类

- `md:` - 中等屏幕及以上（≥768px）
- `hidden md:block` - 移动端隐藏，桌面端显示
- `hidden md:flex` - 移动端隐藏，桌面端 flex 布局
- `w-full md:w-96` - 移动端全宽，桌面端固定宽度

### 组件变体

**ChatNavRail 组件**支持两种变体：
- `variant="desktop"` - 垂直布局，完整导航
- `variant="mobile"` - 水平布局，精简图标

### 状态管理

使用 `useState` 管理移动端侧边栏开关状态：
```typescript
const [isMobileSidebarOpen, setIsMobileSidebarOpen] = useState(false);
```

## 用户体验优化

1. **触摸友好**：按钮尺寸适中，易于点击
2. **自动关闭**：选择项目后自动关闭侧边栏
3. **流畅动画**：Sheet 组件提供平滑的滑入/滑出动画
4. **内容优先**：移动端隐藏次要功能，聚焦核心聊天体验
5. **无遮挡**：顶部导航栏固定，内容区域自动留出空间

## 测试建议

1. 在不同屏幕尺寸下测试（手机、平板、桌面）
2. 测试横屏和竖屏模式
3. 验证触摸交互的响应性
4. 检查长文本的截断和滚动
5. 确认所有功能在移动端可访问

## 未来优化方向

1. 添加手势支持（滑动关闭侧边栏）
2. 优化移动端输入框体验（虚拟键盘适配）
3. 支持 PWA（渐进式 Web 应用）
4. 添加移动端专属的快捷操作
